## 설계 의사결정
- 데이터 소스는 **마크다운 우선**으로 설계하고, Notion은 향후 확장용 어댑터로 분리한다.
- `/about`는 포트폴리오 허브(프로젝트/경력/스킬)로 구성한다.
- 카테고리/태그 탐색은 URL 기반으로 일관된 라우팅 규칙을 유지한다.
- 상세 비즈니스 규칙은 Functional Design 단계(실행 시)에서 다룬다.
