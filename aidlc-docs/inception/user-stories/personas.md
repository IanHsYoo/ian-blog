# 페르소나

## 개요
ian-blog의 주요 사용자 유형을 정의한다. 스토리는 이 페르소나를 기반으로 작성된다.

---

## Persona 1: 기술 블로그 독자 (Developer Reader)

| 항목 | 내용 |
|---|---|
| **이름** | 민수 (개발자) |
| **역할** | 소프트웨어 엔지니어 (3~7년차) |
| **목표** | 기술 아티클을 읽고, 새로운 기술 트렌드와 실무 경험을 학습 |
| **동기** | 실무에 적용 가능한 기술 인사이트, 코드 예제, 아키텍처 패턴 습득 |
| **행동 패턴** | 검색(Google/SNS)으로 유입 → 포스트 읽기 → 관련 포스트 탐색 → 카테고리/태그로 필터링 |
| **디바이스** | 데스크탑(업무 중) 70%, 모바일(출퇴근) 30% |
| **핵심 니즈** | 코드 구문 강조, 읽기 시간 표시, 카테고리별 탐색, 빠른 로딩 |

---

## Persona 2: 채용 담당자 (Recruiter)

| 항목 | 내용 |
|---|---|
| **이름** | 지영 (HR/채용 담당자) |
| **역할** | 기업 채용 담당자 또는 헤드헌터 |
| **목표** | 블로그 작성자의 기술 역량, 경력, 프로젝트 경험을 빠르게 파악 |
| **동기** | 채용 후보자 평가, 기술 수준 확인, 포트폴리오 검토 |
| **행동 패턴** | 링크(이력서/LinkedIn)로 유입 → About/포트폴리오 확인 → 경력 타임라인 → 프로젝트 → 기술 블로그 일부 확인 |
| **디바이스** | 데스크탑 80%, 모바일 20% |
| **핵심 니즈** | 경력 타임라인 한눈에 파악, 프로젝트 목록과 기술스택 확인, 빠른 네비게이션 |

---

## Persona 3: 블로그 작성자 (Author)

| 항목 | 내용 |
|---|---|
| **이름** | Ian (블로그 소유자) |
| **역할** | 블로그 콘텐츠 작성자 및 관리자 |
| **목표** | 기술 지식 공유, 포트폴리오 관리, 개인 브랜딩 |
| **동기** | 학습 기록, 커리어 발전, 커뮤니티 기여 |
| **행동 패턴** | 마크다운으로 포스트 작성 → 프론트매터 설정 → git push → 자동 빌드/배포 |
| **디바이스** | 데스크탑 100% (콘텐츠 작성) |
| **핵심 니즈** | 간편한 콘텐츠 작성/관리, 빠른 빌드, 포트폴리오 데이터 쉬운 업데이트 |

---

| 다크모드 | 편의 | — | — |
